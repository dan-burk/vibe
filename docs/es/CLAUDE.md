# Documentación de Tutoriales Vibe

Este repositorio contiene tutoriales completos y amigables para principiantes sobre el uso de Claude Code, VS Code y herramientas de desarrollo relacionadas. Todos los tutoriales están diseñados para principiantes completos sin experiencia previa en programación.

## Habilidades Disponibles

Las habilidades de Claude Code amplían la funcionalidad con capacidades especializadas:

### Habilidades de Procesamiento de Documentos
- **docx** - Creación, edición y análisis de documentos de Microsoft Word
- **pdf** - Manipulación de PDF, extracción de texto, llenado de formularios y generación
- **pptx** - Creación, edición y análisis de presentaciones de PowerPoint
- **xlsx** - Creación de hojas de cálculo de Excel, análisis, fórmulas y visualización

### Habilidades de Investigación
- **medical-research** - Búsqueda en PubMed y resumen de literatura científica
- **generating-stock-reports** - Investigación de empresas e informes de análisis de acciones

### Habilidades de Análisis de Datos
- **csv-stats** - Estadísticas de archivos CSV y análisis de datos

## Comandos Slash

Comandos personalizados disponibles a través de `.claude/commands/`:

- `/tutorial [topic]` - Crear un nuevo tutorial amigable para principiantes siguiendo la guía de estilo
- `/review-tutorial [file]` - Revisar y pulir un tutorial existente para claridad, precisión y formato
- `/review-translation [file]` - Revisar y actualizar traducciones al japonés para que coincidan con los originales en inglés
- `/bioinfo-newsletter` - Generar un boletín de investigación en bioinformática a partir del contenido del blog

## Directrices de Estilo de Escritura

Todos los tutoriales siguen estos principios:

### Público Objetivo
- Principiantes completos sin experiencia en programación
- Usuarios no técnicos que aprenden herramientas de desarrollo
- Explicaciones claras sin jerga técnica

### Estructura
- Enlace `[Home](./)` en la parte superior
- Gancho atractivo (2-4 oraciones)
- Sección de Conceptos Clave (máximo 3-4 términos)
- Lo Que Necesitará (prerrequisitos + estimación de tiempo)
- Instrucciones paso a paso con formato `## Step N: Action Verb + What`
- Sección de Próximos Pasos (2-3 sugerencias)
- Sección de Solución de Problemas (2-3 problemas comunes)
- Fecha de creación y autor al final

### Formato
- Use viñetas (-) para sub-pasos, no listas numeradas
- Negrita para elementos de interfaz de usuario: **Nombre del Botón**
- Comillas invertidas para comandos: `command`
- Bloques de código para comandos de múltiples líneas
- Párrafos cortos (máximo 2-3 oraciones)
- Instrucciones basadas en menús en lugar de atajos de teclado

### Extensión
- Objetivo de ~400 líneas o menos
- Enfoque en información esencial
- Elimine explicaciones innecesarias
- Manténgalo accionable y conciso

## Traducciones al Japonés

Traducciones completas al japonés están disponibles en la carpeta `ja/`. Todos los tutoriales en inglés tienen versiones en japonés correspondientes con la misma estructura de nombres de archivo:

- Todas las guías de instalación
- Todas las guías de configuración de VS Code
- Todas las guías de programación (R y Python)
- Todas las guías de Git y GitHub
- Todos los tutoriales avanzados
- Proyectos de investigación y escritura

Directrices de traducción:
- Términos técnicos (Git, Docker, VS Code) se mantienen en inglés
- Instrucciones de línea de comandos nunca se traducen
- Elementos de interfaz de usuario coinciden con la localización japonesa cuando es aplicable
- Japonés natural y fluido usando la forma です/ます
- Terminología consistente en todo el documento

## Contribuciones

Al crear o actualizar tutoriales:

1. Use el comando `/tutorial [topic]` para crear nuevos tutoriales
2. Use el comando `/review-tutorial [file]` para revisar tutoriales existentes
3. Siga las directrices de estilo de escritura anteriores
4. Mantenga los tutoriales enfocados y por debajo de 400 líneas
5. Pruebe todos los comandos e instrucciones
6. Incluya solución de problemas para problemas comunes
7. Agregue fecha de creación y atribución de autor
